# Process Log

## 记录规则

- 日常过程记录以中文为主，优先保证记录速度、判断准确性和后续可追溯性。
- 阶段性总结使用中英双语，便于后续直接转化为 portfolio case study 的过程说明、项目总结或反思文字。
- 如果某条记录未来大概率会进入作品集正文、项目页面或申请材料，建议从一开始就补充英文版本。
- requirements 文档保持中英双语同步；process log 不要求每条都双语，但阶段总结必须双语。

- 准备了 React、TypeScript、Vite 和 Tailwind 的基础配置文件。
- 创建了可复用组件，包括导航、项目卡片、案例结构区块和过程展示区块。
- 添加了首页、项目总览页、四个主要项目页、About 页和 404 页的占位结构。

- 已将网站需求文档转换为中英双语格式。
- 英文作为执行依据。
- 中文作为创作意图和方向复核依据。
- 后续需求文档更新需要保持中英双语同步。

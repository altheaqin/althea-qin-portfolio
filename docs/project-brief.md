# Project Brief Overview / 项目简报总说明

## Purpose / 用途

EN:
This folder stores planning documents for the portfolio website and its main case studies. Each project brief should help define project positioning, content needs, process evidence, and how the work should eventually be presented on the site.

CN:
这个文件夹用于存放作品集网站及其主要案例项目的规划文档。每一份项目简报都应帮助明确项目定位、内容需求、过程证据，以及未来在网站中的呈现方式。

## Writing Rule / 写作规则

EN:
All project briefs should be written in bilingual EN/CN format. English should function as the main execution reference for Codex and as a usable source for future website copy. Chinese should explain the creative intention, working logic, and self-review direction. If a brief is updated later, both languages should be updated together unless the user explicitly requests a single-language draft.

CN:
所有 project brief 都应采用中英双语对照格式书写。英文用于作为 Codex 的主要执行依据，也方便后续直接发展为网站文案草稿；中文用于解释创作意图、工作逻辑和自我复核方向。如果未来要更新某份 brief，默认应同步更新中英文两个版本，除非我明确要求只写单语草稿。

## Current Scope / 当前范围

EN:
At the current stage, the brief system should remain flexible, preserve space for process documentation, use neutral placeholders only, and replace those placeholders only with original works and real process material.

CN:
在当前阶段，brief 系统应保持灵活，预留过程文档空间，只使用中性占位内容，并且只能用原创作品和真实过程材料去替换这些占位内容。
